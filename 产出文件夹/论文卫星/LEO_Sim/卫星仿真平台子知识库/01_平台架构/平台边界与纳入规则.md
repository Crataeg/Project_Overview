# 平台边界与纳入规则

## 纳入范围
- `v7proj` 中直接参与卫星仿真平台运行的 MATLAB 代码
- 与代码中技术细节直接相关的论文、开源 PDF 与算法原典
- 与 `DL / UL / E2E / jammer / classification / dashboard` 直接相关的说明页

## 排除范围
- `Obsidian知识图谱/11_卫星正向设计` 的所有内容
- 专利、终端结构、硬件集成、EMC 正向设计、车厂材料
- 无人机项目与外部草库
- 只和业务落地有关、但不直接改变仿真机制的文档

## 判断规则
- 如果一个资料回答的是“仿真平台怎么计算”，收录。
- 如果一个资料回答的是“产品怎么设计才可交付”，不收录。
- 如果一个资料只提供工程背景，而没有和代码模块形成映射，暂不收录。

## 当前平台边界的代码证据
- `LEO_StarNet_EMC_V7_0_Engineering.m:80-81`：构建 `satelliteScenario`
- `LEO_StarNet_EMC_V7_0_Engineering.m:291-341`：`InfoGAN + GA` 最劣搜索
- `LEO_StarNet_EMC_V7_0_Engineering.m:369-400`：`STFT + LeNet` 分类
- `LEO_StarNet_EMC_V7_0_Engineering.m:427` 起：Dashboard
