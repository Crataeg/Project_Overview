# 卫星仿真平台技术栈

## 1. 星座与几何场景
- 在 `LEO_StarNet_EMC_V7_0_Engineering.m:80-102` 构建 `satelliteScenario`、用户地面站、网关地面站、LEO 星座和 jammer 卫星
- 在 `:110-156` 预计算用户、网关、jammer 的 `az / el / range`
- 在 `:161-195` 构造频率复用与简化 ISL 图

## 2. 链路功率预计算
- `:199-279` 预计算 `DL / UL` 的信号功率、CCI、jammer 基底功率和噪声项
- 关键底座：
  - `emcPrecomputeLinkPower.m`
  - `emcBuildUplinkJammerBase.m`
  - `emcBuildLinkModel.m`

## 3. 链路级求解
- `simulateStarNetV7.m` 统一计算：
  - `SINR`
  - `BER`
  - `THR`
  - `BLER`
  - `DOP`
  - `E2E delay`
  - `Event`
- 当前平台是“系统级功率仿真 + 指标代理模型”，不是全波形物理层细仿真

## 4. 最劣场景搜索
- `LEO_StarNet_EMC_V7_0_Engineering.m:291-341`
- `trainOrLoadJammerGAN.m`
- `genJamAggFromG.m`
- `worstCaseObjectiveV7.m`

逻辑是：
- 先训练或加载 `InfoGAN`
- 再用 `GA` 在 `z + c + JamScale` 空间中搜索最差解
- 最后把最差 jammer 包络喂回 `simulateStarNetV7`

## 5. 干扰识别
- `LEO_StarNet_EMC_V7_0_Engineering.m:369-400`
- `getOrTrainLeNetSTFT.m`
- `generateDatasetSimpleSTFT.m`
- `classifyInterferenceTimeline_powerSampler.m`

逻辑是：
- 从功率级生成 IQ 快照
- 计算 STFT 图像
- 用 `LeNet` 做干扰形态识别

## 6. 输出与可视化
- Dashboard：`LEO_StarNet_EMC_V7_0_Engineering.m:427` 起
- 摘要：`emcWriteSummaryTextV7.m`
- 结果：`outputs_v7/result_v7.mat`
- 配置：`outputs_v7/cfg_resolved_v7.mat`
